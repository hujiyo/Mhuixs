# Mhuixs

## 项目介绍

Mhuixs是一个基于内存的数据库，旨在结合关系型与非关系型数据库的优势，提供国产化、可集成化、丰富的数据结构支持以及高级的安全特性。Mhuixs支持键值对基于MurmurHash算法进行分列扩散，以实现高效的数据查询和管理。此外，Mhuixs还支持键关系的基础创建、键间联系强度的设定，以及软多线程命令的事务处理等高级功能。

## 项目背景

随着信息化战争的趋势，基于内存的高速数据库将成为未来主流。Mhuixs以其独特的优势，包括：

- 支持关系型与非关系型数据管理
- 国产化
- 可集成化，能够内嵌到其他程序中
- 丰富的数据结构支持
- 高级安全特性
- AI与人类权限分级管理

## 项目结构

### 开发环境设置

确保编辑器Mhuixs/为项目根目录，以避免头文件包含错误。

### 文件目录结构

项目文件目录结构如下：

- `datstrc`：数据结构文件夹，包含表、队列等数据组织方式的相关文件。
- `mlibs`：存放最基础的库文件，如Mhudef.h（基本参数定义文件）和hook.h（链接不同数据结构的结构）。

......等等未来会加更多

每个文件夹内都包含一个`文件夹名-dir介绍.txt`文件，描述本文件夹的作用和内容。

## 获得项目

安装配置Git后：

1. 初始化本地仓库：`git init`
2. 添加远程仓库：`git remote add origin <仓库URL>`
3. 拉取远程仓库代码：`git fetch origin`
4. 合并远程分支：`git merge origin/<分支名>`
5. 切换到远程分支并拉取：`git checkout <分支名>` 和 `git pull origin <分支名>`

请替换`<仓库URL>`和`<分支名>`为实际的仓库地址和分支名称。
